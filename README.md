## 📅 Desafíos día por día

### 🌠 Día 1 – Creación de repositorio en GitHub
- Crear repositorio desde template.
- Agregar un README (opcional).  
- Editar la descripción del perfil.  

### 🌠 Día 2 – Acciones: copiar/pegar, commits y subir archivos en GitHub
- Parte 1: a. Crear los archivos HTML y CSS del template (copiando y pegando el código manualmente). 
           b. Realizar el commit desde la página.
- Parte 2: a. Subir una carpeta llamada images con las 4 imágenes del template.
           b. Opcional: Usar Upload files para subir todo.

### 🌠 Día 3 – Clonación del repositorio remoto para editarlo localmente en VSC
- Instalar VSC y Git.  
- Clonar repositorio en VSC a partir de la URL disponible en GitHub.   

### 🌠 Día 4 – Personalización de index.html (3 películas distintas) y ejecución del correspondiente commit
- Parte 1: Modificar nombre, imagen, calificación y descripción de las películas en el index.html.
- Parte 2: Realizar commit de los cambios a través de la pestaña "Source Control" de VSC.     

### 🌠 Día 5 – Cómo revertir un commit cuando algo sale mal
- Parte 1: Romper el código a propósito en index.html (modificar, hacer commit y push del cambio)
- Parte 2: Revertir el commit para eliminar el error (usar el comando rever [id del commit roto])   

### 🌠 Día 6 – Resolución de un conflicto de merge 
- Parte 1: a. Modificar una parte del código de index.html, directamente en GitHub (descripción de la tercera     
  película).  
           b. Commit del cambio.    
- Parte 2: a. Editar la misma parte del código desde VSC (pero con otro texto).  
           b. Commit del cambio. 
- Parte 3: a. Realizar un pull a través de Source Control.
           b. Resolver el conflicto de merge 
           c. Commitear el resultado del merge y sincronizar ambos repositorios.

### 🌠 Día 7 – Publicación del proyecto en GitHub Pages
- Dentro de la cuenta de GitHub, seleccionar el repositorio y publicar la página en GitHub Pages .  
- Visualizarla a través del enlace proporcionado por GitHub.

## 🎬 Proyecto de películas
El desafío se aplicó sobre una **Página web de películas populares**, que sirvió como base para:  
- Practicar commits, push, pull, revert y merge.  
- Publicar en GitHub Pages.  
- Probar funcionalidades de control de versiones en un proyecto real.
